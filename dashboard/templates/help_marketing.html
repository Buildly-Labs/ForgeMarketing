  1. Complete onboarding and confirm at least one active brand.

Lead Radar and Lead Generation

Use Lead Radar for source-driven research, candidate review, and conversion into approved leads.

  1. Define Sources with query keywords and run frequency.
  2. Run source jobs manually or run due sources in batch.
  3. Review Candidates, approve/reject/mark duplicates, and convert to leads.
  4. Track Leads, regions, and follow-up tasks.

API and Integration

Use API endpoints for external orchestration and scripted operations.

  • Status and health checks: /api/status
  • Lead Radar automation: /api/lead-radar/research-jobs/run and /api/lead-radar/research-jobs/run-due
  • Lead lifecycle: /api/leads and /api/leads/regions

Troubleshooting

  • If nav links fail at root paths, use /marketing/* routes and confirm gateway redirects are current.
  • If dropdown menus close unexpectedly, refresh cache after deployment of the shared base template fix.
  • If Lead Radar finds no candidates, verify source query keywords, source type, and run frequency.
  • If scheduled tasks fail, check task type configuration and command support in runner services.
  • If analytics are empty, verify credential setup and date ranges in reports views.
